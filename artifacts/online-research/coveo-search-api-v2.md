# Coveo Search API v2

Note: docs.coveo.com was returning HTTP 503 at time of research (2026-06-24). Field definitions are sourced from the IQuery interface reference (coveo.github.io/search-ui), the go-coveo library struct definitions, the Headless SDK Result interface, and web search corroboration of field names in official documentation. These are authoritative Coveo-published sources.

## Endpoint

```
POST https://platform.cloud.coveo.com/rest/search/v2
```

Regional variants:
- EU: `https://eu.platform.cloud.coveo.com/rest/search/v2`
- APAC: `https://au.platform.cloud.coveo.com/rest/search/v2`

Organization-scoped URL pattern: `https://{organizationId}.org.coveo.com/rest/search/v2`

## Authentication

```
Authorization: Bearer <token>
Content-Type: application/json
Accept: application/json
```

The token is either an API key or a search token (short-lived, user-scoped, obtained via the Search Token endpoint).

## Request Body Fields

All fields are optional. Omitting `q` is valid (returns all results ordered by default ranking).

| Field | Type | Description |
|-------|------|-------------|
| `q` | string | Basic query expression — the user's search keywords |
| `aq` | string | Advanced query expression — programmatically appended query clauses (e.g. field filters generated by facets) |
| `cq` | string | Constant query expression — clauses constant for all users; cached separately by the index (e.g. permission filters) |
| `lq` | string | Contextual text passed to Coveo ML for intelligent term extraction; not surfaced in results directly |
| `dq` | string | Disjunction query — merged with other expression parts using OR |
| `numberOfResults` | number | Number of results to return; defaults to 10 |
| `firstResult` | number | Zero-based index of the first result (pagination offset); defaults to 0 |
| `searchHub` | string | Identifies the search hub (search interface context); used for analytics and query pipeline routing |
| `tab` | string | Active tab identifier; used for query pipeline routing |
| `locale` | string | BCP-47 locale code e.g. "en-US"; influences language-specific ranking and stemming |
| `timezone` | string | IANA timezone name; affects date-relative filters |
| `pipeline` | string | Explicitly names the query pipeline to use; defaults to "default" |
| `sortCriteria` | string | Sort expression. Values: `relevancy`, `dateascending`, `datedescending`, `qre`, `nosort`, `@fieldname ascending`, `@fieldname descending` |
| `excerptLength` | number | Maximum character length of generated excerpts; defaults to 200 |
| `enableQuerySyntax` | boolean | Whether to interpret advanced Coveo query syntax in `q` |
| `debug` | boolean | When true, includes `rankingInfo` in each result |
| `groupBy` | array | Facet/grouping requests — array of GroupByRequest objects |
| `context` | object | Key-value context object for query pipeline conditions and ML context |
| `facets` | array | Coveo Headless-style facet request objects |

## Response Shape

Top-level fields:

| Field | Type | Description |
|-------|------|-------------|
| `totalCount` | number | Total number of matching results in the index |
| `totalCountFiltered` | number | Total after applying index permissions |
| `duration` | number | Total query duration (ms) |
| `indexDuration` | number | Time spent in the index (ms) |
| `requestDuration` | number | Network time (ms) |
| `queryUid` | string | Unique identifier for this query execution; used in analytics calls |
| `searchUid` | string | Alias for queryUid in some versions |
| `results` | array | Array of result items (see below) |
| `groupByResults` | array | Facet/groupby results |
| `pipeline` | string | Name of the query pipeline that processed the request |
| `splitTestRun` | string | A/B test run name if applicable |

## Result Item Shape (results[])

| Field | Type | Description |
|-------|------|-------------|
| `title` | string | Title of the item |
| `uri` | string | URI of the item in the index; avoid using this for hyperlinks |
| `clickUri` | string | The hyperlinkable URI — use this for `href` attributes |
| `printableUri` | string | Human-readable URI/path for display |
| `uniqueId` | string | Opaque unique identifier for the item; required for certain API calls (e.g. quickview) |
| `excerpt` | string | Contextual snippet generated by the index; may be empty for binary content |
| `firstSentences` | string | First sentences of the item |
| `summary` | string | Item summary field (may be null) |
| `score` | number | Total ranking score |
| `percentScore` | number | Score expressed as a percentage |
| `rankingInfo` | string or null | Detailed ranking debug information; only present when `debug: true` in the request |
| `rankingModifier` | string | Type of ranking modification applied (e.g. "Featured Result") |
| `raw` | object | Key-value map of all raw index field values for the item; field names are index-defined (e.g. `raw.sfid`, `raw.source`, `raw.date`) |
| `flags` | string | Semicolon-separated index flags for the item |
| `absentTerms` | string[] | Query terms not matched by this result |
| `hasHtmlVersion` | boolean | Whether the item has an HTML quick-view version |
| `isTopResult` | boolean | Whether the item is a Featured Result from the query pipeline |
| `isRecommendation` | boolean | Whether the item was returned by a Coveo ML recommendation model |
| `titleHighlights` | HighlightKeyword[] | Word offsets for highlighting query terms in `title` |
| `excerptHighlights` | HighlightKeyword[] | Word offsets for highlighting query terms in `excerpt` |
| `printableUriHighlights` | HighlightKeyword[] | Word offsets for highlighting in `printableUri` |
| `firstSentencesHighlights` | HighlightKeyword[] | Word offsets for highlighting in `firstSentences` |

## raw field

`raw` is a plain object. Its keys are the index field names configured in the Coveo organization. Common fields vary by content source but typically include things like:
- `raw.date` — indexed date
- `raw.source` — source name
- `raw.sourcetype` — source connector type
- `raw.size` — document size
- `raw.filetype` — file type

For Salesforce Help content the raw fields will be Salesforce/Coveo-specific and discoverable by inspecting a live response.
