# Human Prose Editing Reference

This reference supports the `human-prose-editing` skill with local paragraph-level editing guidance and rewrite examples.

## Source Basis

This guidance is distilled from style, concision, and plain-language editing practice for technical prose.

- Style guidance for sentence energy and specificity
- Concision guidance for cutting repeated meaning
- Plain-language guidance for direct wording
- Technical style guidance for voice and strong verbs

## Paragraph Method

1. Identify the one point the paragraph should make.
2. Cut sentences that only restate that point.
3. Move the sharpest sentence earlier if the paragraph starts slowly.
4. Replace general claims with a concrete behavior, file, API, or consequence.
5. End on an insight, implication, or caution.

## Rewrite Example

Before:

"When it comes to promotions, SFCC offers a powerful and flexible mechanism that is important for modern ecommerce experiences."

After:

"SFCC promotions let merchandisers change pricing behavior without editing storefront code for each campaign. That separation matters because pricing rules often change faster than a deployment cycle."

Before:

"It is worth noting that caching can have a significant impact in a variety of situations."

After:

"Caching matters here because the same rendered page can be reused across requests. If a template mixes shopper-specific data into cached output, one shopper can see another shopper's state."

## Review Questions

- Does every paragraph make one point?
- Are repeated claims collapsed into one stronger statement?
- Does the prose name real things instead of abstractions?
- Does the paragraph end with insight instead of fade-out summary?
